Business View

What The Application Delivers

Operational Visibility

Gives the team a shared view of rota coverage, leave status, weekly staffing, and current employee availability from one place.

Controlled Editing

Keeps rota editing inside admin mode while regular viewers stay in read-only mode. Admin credentials for Edit ROTA come from Firestore `users/admin`.

Operations Workspace

Pulls supporting tools into the same experience through Jira panels and quick links to APF, SFTP, Certificate, and Documentation microsites.

System Design

Runtime Architecture

The application is primarily a browser-side React app. It reads and writes rota data directly with Firestore, routes Jira queries through a Netlify Function, and reaches APF, SFTP, Documentation, and Certificate records through Oracle-backed microsite APIs.

Why Firestore is used directly

Rota data, employee order, and audit logs are all small enough to work well with real-time Firestore listeners from the frontend.

Why Jira and Oracle stay server-side

Jira credentials and Oracle database access stay on the server side through Netlify functions and APF Node services, keeping the browser free of direct secrets.

Data Model

Storage Model And Responsibilities

Collection / Document Purpose How The App Uses It
users/{username} Employee master data and ordering metadata Used to build the employee list; `users/admin.password` is used for Edit ROTA admin mode
rota/master Central rota document Stores keyed assignments like year-month-week-employee-day with shift and leave data
logs/{logId} Audit trail Written whenever admin updates rota cells; shown on the Logs page
config/admin Optional backup admin configuration Still supported as a secondary fallback if `users/admin` is unavailable

Firestore Scope

Firestore remains the live source of truth for the rota application itself, including employee ordering, rota assignments, admin access, and audit logging.

Oracle Microsite Scope

Oracle now stores APF links, SFTP partner rows, Documentation records, and tracked Certificate entries. APF, SFTP, and Certificate also support Excel template download plus bulk publish flows.

Access Flow

Portal Login And Edit ROTA Admin Flow

Portal Access

  • Landing page asks for the default portal credentials.
  • Successful login creates a local browser session.
  • User enters the main app and can view dashboard features.

Edit ROTA Admin Access

  • Clicking the MyRota logo opens the admin modal.
  • The app reads `users/admin` from Firestore.
  • If the entered password matches, editing and logs are enabled.

Integration Layer

Jira Dashboard Flow

Frontend behavior

Home page lazily loads the Jira dashboard panel. Clicking a metric opens a dedicated issue-detail screen backed by the same Netlify function.

Backend behavior

The Netlify function validates environment variables, runs Jira JQL queries, normalizes issue payloads, and returns summary or detail JSON to the frontend.

Delivery Model

Build And Deployment Pipeline

Publish folder

Netlify serves `dist/` and exposes the Jira function from `netlify/functions`, while the Oracle-backed APF microsites can also run through their Node host.

Branch flow

Source code is committed to `main`, pushed to GitHub, and deployed by Netlify.

Operational dependency

Live Jira dashboards require correctly configured Netlify environment variables, and the linked microsites require Oracle connectivity when run from APF services.

Repository View

Main Folders And Ownership

Path Role Notes
src/ Main MyRota frontend Primary working area for dashboard, admin mode, reports, notifications, and Jira UI
src/jira/ Jira frontend integration API client, summary panel, and issue detail page
netlify/functions/ Serverless endpoints Current production Jira backend entrypoint
netlify/jira/ Jira query helpers JQL definitions, payload shaping, environment handling
scripts/ Build support Copies microsites into the publish folder after the Vite build
APF/ Linked microsites and Oracle services Contains APF, SFTP, Documentation, and Certificate workspaces plus shared Oracle helpers
docs/ Documentation set Contains this presentation and the detailed technical document

Recommended maintenance boundary

Treat `src/`, `src/jira/`, `netlify/`, `scripts/`, `docs/`, and the APF Oracle entry points as the primary MyRota ownership zone. The larger generated assets should be refreshed only when the portal build or linked microsites are intentionally updated.
